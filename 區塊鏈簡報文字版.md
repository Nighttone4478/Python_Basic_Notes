# 🧠 AI 模型訓練與應用 QA

**Q1:** 請問訓練一個AI模型需要花費多少時間？  
**A1:** 這個問題沒有一個肯定的答案，因為這取決於你給予甚麼任務給這個模型。如果說是小型專案的話可能一天內就能完成了，如果是比較大型的就要數天。  

**Q2:** 自動駕駛技術當前還是存在著可能有車禍的風險，請問該如何去改善？  
**A2:** 制定明確的法規來規範自動駕駛車輛的駕駛規章，以及使用更高級的模組去搭配整台自駕車。  

**Q3:** 除了硬體提升，還有哪些因素使AI模型的訓練效率大幅提高？  
**A3:** 除硬體外，演算法優化（如深度學習框架的高效實現）、分布式運算、多GPU並行訓練、以及更好的資料預處理與批量化技術，也大幅提升了AI模型訓練的效率。

# 🔒 資安議題 QA

**Q1:** 如果我的電腦被破解，重灌有辦法解決問題嗎？  
**A1:** 重灌可以解決，但是如果真的不放心可以買一台新電腦。  

**Q:** 能以資安專家的角度提供一些保護資安的建議嗎？  
**A2:** 不要亂下載來路不明的檔案，如果有陌生郵件也不要亂點。  

**Q3:** 這個攻擊手段跟勒索軟體類似嗎？  
**A3:** 勒索軟體比較偏向於利用釣魚手段來進行攻擊，而不是像本次攻擊手法般，利用產品漏洞進行攻擊。  

**Q4:** 什麼是RCE？  
**A4:** RCE(Remote Code Execution)白話意思就是你可以對受害者電腦進行任意操作。  

**Q5:** 是否所使用的後端框架是有開源的話，是否比較容易被攻擊呢？是否可以解讀成程式碼不開源就會比較安全呢？  
**A5:** 不一定，也是有非開源的後端框架漏洞很多，因為駭客可以透過Fuzzing(模糊測試)的方式用來測試是否有可疑的行為，再進一步的攻擊。

# 🎮 遊戲與 AI QA

**Q1:** 主流遊戲引擎有哪些？  
**A1:** Unity, Unreal Engine, Godot.  

**Q2:** AI在遊戲中的現有應用主要包括哪些方面？  
**A2:** NPC行為、內容生成、個人化體驗。  

**Q3:** 遊戲與AI結合的未來發展方向可能有哪些？  
**A3:** 生成式AI劇情。  

**Q4:** 遊戲開發中整合AI可能面臨的挑戰與問題是什麼？  
**A4:** 安全漏洞風險隨科技發展增加，需保護財務與機密資訊。  

**Q5:** AI如何提升遊戲的個人化體驗和內容生成？  
**A5:** 根據玩家行為動態調整難度與劇情，提供定制化體驗。  

**Q6:** 前面有介紹到下prompt給AI然後生出內容的遊戲，那有考慮過prompt injection的問題嗎？  
**A6:** Prompt Injection在遊戲中可能導致規則破壞或不當內容生成，需透過輸入過濾、上下文隔離、輸出限制及監控來防範，平衡創意與安全。  

**Q7:** 使用AI工具會不會導致未來的遊戲看起來、玩起來都越來越像？該如何避免AI扼殺創意多樣性？  
**A7:** AI工具可能使遊戲同質化，需鼓勵原創設計、限制模板依賴、融入玩家反饋，保持創意多樣性。

# ⛓️ 區塊鏈技術 QA

**Q1:** 總共有哪幾種鏈？  
**A1:** 四種，分別為公有鏈、私有鏈、聯盟鏈、混合鏈。  

**Q2:** 區塊鏈具備什麼優勢？  
**A2:** 去中心化、不可篡改性、透明與可追溯、安全性高、自動化與效率提升、信任機制的建立。  

**Q3:** 運作流程是什麼？  
**A3:** 記錄交易、達成共識、連結區塊、分享總帳。  

**Q4:** 區塊鏈的哈希值是一段蠻冗長的亂碼，那在現今那麼進步的情況下，這個哈希值在現今或未來是否有可能被破解？  
**A4:** 目前幾乎不可能。哈希算法像是一個「單向的碎紙機」，再強也得花上幾千萬年以上才能反推出原始資料。除非量子電腦成熟，才有「理論上」破解的可能，但目前還早。  

**Q5:** 區塊鏈有沒有遭遇過什麼重大的資安問題？  
**A5:** 有，但問題多半出在「應用層」而不是「區塊鏈本身」。例如交易所被駭、私鑰外洩、智能合約漏洞等。區塊鏈的底層機制本身幾乎沒被直接攻破過。  

**Q6:** 台灣在區塊鏈的應用是數位身分證，但是政府對於加密貨幣保持著高度監察，為何會有這麼矛盾的態度？  
**A6:** 因為政府重視「技術」但不信任「投機市場」。區塊鏈技術能提升效率與安全，但加密貨幣涉及金流、洗錢等風險，所以政府選擇應用技術、但嚴管貨幣。  

**Q7:** 區塊鏈那麼的方便為什麼不取代傳統銀行？  
**A7:** 因為區塊鏈雖然去中心化、透明又安全，但缺點是交易速度慢、能量消耗高，且缺乏法律監管。銀行除了轉帳外，還有貸款、保險、利息管理等功能，這些目前區塊鏈還沒辦法完全取代。  

**Q8:** 挖礦是指驗證交易的過程，那第一筆交易是怎麼出現的？  
**A8:** 第一筆交易叫「創世區塊（Genesis Block）」 ，是由中本聰在比特幣系統啟動時手動寫入的，當時的獎勵是50BTC。那筆交易沒有前一個區塊，它是整個區塊鏈的起點。  

**Q9:** PoW與PoS是透過競爭的方式取得建立區塊的機會，那在私有鏈與聯盟鏈這種環境中可能都是受信任的節點的情況下，是否有更適合的共識機制？或是不需要共識機制？  
**A9:** 在私有鏈或聯盟鏈中，因為節點本身可信，所以不需要PoW或PoS這種競爭機制，而是改用PBFT或Raft式，速度更快也更節能。

# 🧩 遊戲引擎架構 QA

**Q1:** 遊戲引擎是用來做什麼的？對開發者有什麼好處？  
**A1:** 遊戲引擎是開發遊戲時使用的工具箱，總合了開發遊戲時會用到最常用的各項工具，並且將開發過程簡易與視覺化，讓開發過程更簡單便捷。  

**Q2:** 遊戲引擎分為幾項基礎架構？  
**A2:** 七大項，分別是渲染系統、音訊系統、動畫系統、物理引擎、腳本引擎、資源管理及平台支援。  

**Q3:** 遊戲引擎七大項分為哪兩類？這兩類分別代表什麼？  
**A3:** 遊戲引擎主要分為前端和後端。前端是我們所能看到和聽到的影音功能，後端則是包含遊戲實際運行的相關程式執行。

# ✋ 手語與 AI 辨識 QA

**Q1:** 手機具有能打字來做溝通，比起手語來更加方便普及，那麼手語翻譯機還有必要嗎？  
**A1:** 手語並非只有應用在聽障人士身上，特殊生（如自閉症）也會用到手語，他們不一定會/方便打字，相反的，他們不一定會認識字。跟我們人說話一樣，能講話我們不會打字，對聽障人士來說，能使用手語就不要用打字（手語效率比較高），為了實踐科技平權，我覺得有必要實現這樣的技術。有些場合高效率的溝通是必要的，如開會、上課。這點探討到一個認同的問題，今天如果我們大力推崇使用文字，這樣間接的否定手語存在的必要性。  

**Q2:** 請問為什麼已經有文字了，還需要使用手語呢？  
**A2:** 手語更具有即時性、也可以協助特殊需求生理解句子中的內容。  

**Q3:** 請問從無到有產生一個AI模型需要經過哪4個步驟？  
**A3:** 搜集資料、特徵提取、模型訓練、模型測試。  

**Q4:** 請問台灣在手語辨識方面有什麼限制呢？  
**A4:** 資料庫太少。  

**Q5:** 如何讓AI理解手語語意而非僅辨識手勢？  
**A5:** AI在辨識出一個手語單字後可以使用NLP訓練框架進行語意辨識。AI能辨識出一個單字也能辨識出一個句子，差別在用的模型不同，以及訓練方法會不太一樣。  

**Q6:** 此項技術在應用上相比傳統文字顯示優勢在哪裡？  
**A6:** 手語具有即時性，同時對於自閉症患者來說，手語更直覺，能夠協助他們了解句子中的重點。  

**Q7:** 此項技術的主要服務人群？  
**A7:** 聽障人士和特殊需求學生以及其親朋好友。先天性的聽障人士沒辦法精準發音，所以相較於口說，手語是較為精準的溝通方式，相較於打字，手語會更為快速。  

**Q8:** 此研究是針對台灣手語（TSL）還是像美國手語（ASL）或英國手語（BSL）這類的手語？若系統是用特定手語訓練的，那它能辨識其他手語嗎？還是需要重新訓練？  
**A8:** TSL，在訓練模型時若含有BSL或ASL，也可以針對這兩種手語進行辨識，但一般來說，TSL、ASL、BSL分開訓練比較好。
